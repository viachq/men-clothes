# Діаграма прецедентів (Use Case)

Актори системи: **Гість** (неавтентифікований), **Клієнт**, **Менеджер**,
**Системний адміністратор**. Ролі ієрархічні: системний адміністратор
успадковує можливості менеджера, менеджер — клієнта.

```mermaid
flowchart LR
    Guest(["Гість"])
    Client(["Клієнт"])
    Manager(["Менеджер"])
    Admin(["Системний адміністратор"])

    subgraph S["Інтернет-магазин чоловічого одягу"]
        UC1(["Перегляд каталогу товарів"])
        UC2(["Пошук / фільтрація товарів"])
        UC3(["Перегляд відгуків"])
        UC4(["Реєстрація"])
        UC5(["Підтвердження email"])
        UC6(["Вхід / вихід"])
        UC7(["Керування профілем"])
        UC8(["Керування кошиком"])
        UC9(["Оформлення замовлення"])
        UC10(["Застосування промокоду"])
        UC11(["Оплата (LiqPay)"])
        UC12(["Перегляд історії замовлень"])
        UC13(["Скасування замовлення"])
        UC14(["Залишення відгуку"])
        UC15(["Керування товарами / категоріями"])
        UC16(["Керування розмірами та залишками"])
        UC17(["Керування замовленнями (статуси)"])
        UC18(["Керування промокодами"])
        UC19(["Перегляд аналітики"])
        UC20(["Модерація відгуків"])
        UC21(["Керування користувачами / ролями"])
    end

    Guest --> UC1
    Guest --> UC2
    Guest --> UC3
    Guest --> UC4
    Guest --> UC5
    Guest --> UC6

    Client --> UC7
    Client --> UC8
    Client --> UC9
    Client --> UC10
    Client --> UC11
    Client --> UC12
    Client --> UC13
    Client --> UC14
    Client --> UC1

    Manager --> UC15
    Manager --> UC16
    Manager --> UC17
    Manager --> UC18
    Manager --> UC19
    Manager --> UC20

    Admin --> UC21
    Admin --> UC15
    Admin --> UC19

    UC9 -.includes.-> UC11
    UC4 -.includes.-> UC5
    UC9 -.extends.-> UC10
```

**include**: оформлення замовлення обов'язково включає оплату; реєстрація
включає підтвердження email. **extend**: застосування промокоду — необов'язкове
розширення сценарію оформлення замовлення.
